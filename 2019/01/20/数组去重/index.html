数组去重

发表于 2019-01-20 | 分类于 js

概述：

本文主要介绍了数组去重的几种简单方式

1. 最基本的去重方式

说明：新建一个数组，遍历原数组，新数组中没有相同的值就把原数组的值push进去，返回新数组。
注意：IE8以下不支持数组的indexof方法

执行结果：[1, 2, 4, 9, 6, 7, 5, 3]

2. 先排序，再去重

说明：使用数组的sort方法给数组排序，排序后相同的值相邻，遍历数组，只需要判断每一项的值是否和前一项的值相同，不同就push到新数组中。
注意：此方法会打乱原来数组的顺序

执行结果:
使用sort方法排序后的数组 [1, “2”, 2, 2, 3, 4, 5, 5, 6, 9, “a”, “a”]
去重后的新数组 [1, “2”, 2, 3, 4, 5, 6, 9, “a”]

3. 利用下标查询

说明：遍历数组，如果当前数组的第i项在当前数组中第一次出现的位置不是i，表示第i项是重复的，忽略掉。否则，存入新数组中。
注意：还是得调用indexOf方法，性能和方法1差不多

执行结果： [1, 2, “2”, 4, 9, “a”, 3, 5, 6]

4.优化遍历数组法(推荐的方法)

说明：利用双重for循环，遍历数组。获取没重复的最后值放入到新数组中。(检测到有重复的值时终止当前循环同时进入外层循环的下一轮判断)

执行结果：
数组中元素的索引 [0, 2, 5, 6, 7]
去重后的数组 [1, 2, 3, 6, 5]

5.对象键值法去重

说明：该方法执行的速度比其他任何方法都快， 就是占用的内存大一些。
新建一js对象以及新数组，遍历传入数组时，判断值是否为js对象的键，
不是的话给对象新增该键并放入新数组。
注意：
判断是否为js对象键时，会自动对传入的键执行“toString()”，
不同的键可能会被误认为一样，例如n[val]– n[1]、n[“1”]；
解决上述问题还是得调用“indexOf”。

执行结果：
对象的结果
{1: Array(1), 2: Array(2), 3: Array(1), 4: Array(1), 5: Array(1), 6: Array(1), 9: Array(1), a: Array(1)}
1: [“number”]
2: (2) [“number”, “string”]
3: [“number”]
4: [“number”]
5: [“number”]
6: [“number”]
9: [“number”]
a: [“string”]
proto: Object
去重后的数组 [1, 2, “2”, 4, 9, “a”, 3, 5, 6]

6.利用数组原型对象上的includes方法

说明：判断新数组中是否包含旧数组中的元素，如果不包含push到新数组中

执行结果：[1, 2, “2”, 4, 9, “a”, 3, 5, 6]

7.利用数组原型对象上的filter和includes方法

说明：利用filter方法过滤掉重复的数组项。

执行结果：[1, 2, “2”, 4, 9, “a”, 3, 5, 6]

8.利用数组原型对象上的forEach和includes方法

说明：利用forEach方法遍历数组，将不重复的数组push到新数组中。

执行结果：[1, 2, “2”, 4, 9, “a”, 3, 5, 6]

9.利用数组原型对象上的splice方法(执行速度最慢)

说明：遍历数组，利用splice方法，将重复的数据项剔除。返回值即为删除后的新数组

执行结果： [1, 2, “2”, 4, 9, “a”, 3, 5, 6]

10.利用Es6的set方法

说明：利用ES6提供的新的数据结构Set。它类似于数组，但是成员的值都是唯一的，没有重复的值。

执行结果： [1, 2, “2”, 4, 9, “a”, 3, 5, 6]
